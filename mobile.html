Layout

doc-box

For the most part .doc-box is inherited from desktop Sassquatch.
It's rounded and bordered and has a white background.
In order for the boxy-ness to be seen in a one-column layout, it also has margins in mobile Sassquatch.

This is a basic doc-box. It has no padding so right now it looks pretty cramped. It usually wraps some kind of content, like .doc-content or .touchList

doc-content

.doc-content is basically just padded content. It can sit directly on the background, like this one is. Believe it or not, this is wrapped in .doc-content.

OR it can be nested within a doc-box for header, footer, and sections:

Slightly complex doc-box

It has .doc-content children to wrap the heading and section.

There are tags for these elements on mobile: m:box, m:section, m:head, m:foot.

fix-flex

This layout pattern is used often in lists when we need to display different content in columns. .ffbox wraps elements that are either .ffbox-flex or .ffbox-fix.

It causes some elements in the row

to flex their width to fill the container, while other elements stay

FIXED

	<div class="ffbox-flex">
		<h4>It causes some elements in the row</h4>
		<p>to flex their width to fill the container, while other elements stay</p>
	</div>
	<div class="ffbox-fix">
		<p>FIXED</p>
	</div>
</div>

Touch Lists

Right now, the touchList class is applying some overrides to the way font-size, margin and padding are defined in sectionLists and dividedLists

<%-- JSP --%>

<li:attendee 
	attendee="${rsvp}" 
	listing="${listing}" 
	rsvp_type="${rsvp_type}" 
	id="rsvp_${rsvp.member.memberId }" 
	json='{	"memberId": ... }'
/>

<li class="touchList-item touchList-item--attendee">
