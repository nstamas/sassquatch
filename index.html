Let's Get Started

Markup first, style second

  • Mark things up semantically and the styles should follow.
  • In general, text should not live naked inside a <div> unless you're doing something 1%-ish.

Occupy stylesheet

  • The default styles should work for the 99% of elements on your page.
  • If an element is truly part of the 1%, wearing a monocle and standing on a yacht, use modifier classes and page-specific styles.

Blocks that fit together

  • Padding (top, right, left) happens on wrappers.
  • Margin (bottom) happens on block-level content.

Box and sections

.doc-box is the main box of the page. 99% of the time, there should be only one.

Optionally, your box can be divided into columns by using the standard grid, if you so desire. Do not use gutters here.

Next, a "section" of content is created by adding .doc-content . Simple boxes may have only one section.

Forms

Buttons

As a rule, buttons are for actions, not navigation. This style applies to 

<input type="submit" />, <button>
, and anything with the class .button.

The default is a simple gray button. There is also red .primary class for "do this one thing" actions.

For side-by-side inputs and buttons, you'll probably want something a little smaller than the default button. For this, you can use the .small class.

Javascript degradation helpers

Why show someone a feature that they can't use without javascript? Why hide javascript-dependent content from people without javascript? These classes will give you a bit more control over how things work with and without JS. No need to utilize the .hasJS class in your stylesheets; you can just use these helpers.

css class purpose
.hide-if-js sets display to "none" if javascript is available
.block-if-js sets display to "block" if javascript is available
.inline-if-js sets display to "inline" if javascript is available
.inline-block-if-js sets display to "inline-block" if javascript is available

Meetup vs. Meetup Group

  • If specifically the group: Meetup Group
  • If specifically the event: Meetup
  • If distinction isn't important: Meetup
  • If a generic verb: Let's meetup and…

Your vs. my

Let the user have a conversation with the interface. Don't speak for the user.

  • Your Meetups
  • My Meetups

We

Who's we? There aren't elves in the computer doing work. Let the user be the active party.

  • Send a message to the Organizer
  • Click here and we'll send this message to the Organizer

Capitalization

  • Sentence case is the way
  • Title Case Is Not The Way

Style

  • Keep it short. Keep it snappy.
  • Can you stand up and exclaim loudly what you've written without an ironic grin? If not, use a period, not an exclamation mark.
